## T1 — Dépendance + composant carte réutilisable

### Objectif
Un seul composant carte, utilisé par l'annuaire et la fiche.

### Décision clé
- **Deps runtime** : `leaflet` + `react-leaflet` (+ `@types/leaflet` en devDep). Importer le **CSS de
  Leaflet** (`leaflet/dist/leaflet.css`).
- **Piège connu des icônes** : le marqueur par défaut de Leaflet casse avec les bundlers (chemins
  d'images). **Utiliser un `divIcon` maison** (petit marqueur CSS/SVG) plutôt que l'icône PNG par
  défaut — évite le bug et reste dans le design system. Marqueurs distincts : **MSP** (repère fixe),
  **référence patient** (si active), **fiche(s)**.
- **Tuiles** : `TileLayer` OSM standard, avec l'**attribution obligatoire** « © OpenStreetMap
  contributors ». (Nuance RGPD assumée, cf. cadrage.)

### Étapes
1. `npm i leaflet react-leaflet` + `npm i -D @types/leaflet`. Vérifier que le **build ne gonfle pas
   déraisonnablement** ; si besoin, charger la carte en **import dynamique** (`React.lazy`) pour ne pas
   l'embarquer dans le bundle initial de l'annuaire.
2. Créer `src/components/Map/ProximityMap.tsx` :
   - props : `points: { id: string; coords: LatLng; label: string; kind: 'msp'|'reference'|'contact' }[]`,
     `activeId?`, `onSelect?(id)`, `fitToPoints?: boolean`.
   - rend une carte Leaflet, cadrée automatiquement sur l'ensemble des points (`fitBounds`), marqueurs
     par `kind`, popup au clic (label). Hauteur/rayon via tokens.
   - **Aucun point → ne pas afficher la carte** (ou message « position à préciser »).

## T2 — Carte partagée sur l'annuaire

### Objectif
Une carte unique montrant tous les résultats filtrés + la MSP + la référence active.

### Décision clé (perf)
**Une seule carte pour tout l'écran**, pas une mini-carte par ligne (cf. cadrage — 1000 cartes =
proscrit). La carte lit **les résultats déjà filtrés** (mêmes données que la liste).

### Étapes
1. Dans `AnnuairePage.tsx`, ajouter un **panneau carte repliable** (bouton « Afficher la carte ») qui
   passe à `ProximityMap` : les contacts filtrés **ayant des coordonnées**, + `MSP_COORDS`, + la
   référence active si ≠ MSP. Contacts sans coordonnées → **exclus de la carte** (comptés dans un petit
   « N sans position »).
2. **Lien carte ↔ liste** : `activeId`/`onSelect` — cliquer une épingle surligne/scrolle la ligne ;
   survoler/sélectionner une ligne peut centrer la carte (au minimum : épingle → ligne).
3. Replié par défaut sur mobile (la liste prime — cf. ARCHITECTURE §Contraintes UI).

## T3 — Carte sur la fiche

### Étapes
Dans `FichePage.tsx`, après `CoordsBlock`, insérer un bloc carte **si** la fiche a des coordonnées :
`ProximityMap` avec deux points (fiche + MSP), la **distance** affichée en clair
(`formatDistance(haversineKm(coords, MSP_COORDS))`, ou depuis la référence active). Pas de coordonnées
→ afficher « Position à préciser » (invite discrète à compléter l'adresse), pas de carte vide.
