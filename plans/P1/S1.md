(`ARCHITECTURE.md` §Maquette).

3. `global.css` : exposer ces couleurs en variables CSS, reset (`*{box-sizing}`, `body{margin:0}`),

4. Importer `@fontsource/plus-jakarta-sans/{400,500,600,700,800}.css` + `global.css` dans `main.tsx`.

## T2 — Routing + Layout + pages stub

### Objectif
Mettre en place `react-router-dom`, la barre de navigation (maquette) et une page stub par écran, pour
que le routing soit **définitif** (S3–S7 ne touchent que leur propre dossier de feature).

### Décision clé
Routes finales : `/connexion` · `/` (annuaire) · `/contact/:id` (fiche) · `/nouveau` · `/contact/:id/modifier`
· `/impression` · `/membres`. Layout = barre du haut (dégradé teal→bleu) avec logo « MSP Ménilmontant »,
pills de nav, indicateur « N sélectionné(s) → Imprimer » (props, pas de state global ici), « + Ajouter »,
pastille profil. La barre est un composant présentational ; l'état de sélection viendra plus tard (props).

### Lire / Modifier
- Créer `src/app/router.tsx` (createBrowserRouter + routes → pages stub), `src/app/Layout.tsx` (TopBar + `<Outlet/>`).
- Créer les **pages stub** (placeholder « À câbler ») dans leurs dossiers de feature :
  `src/features/annuaire/AnnuairePage.tsx`, `src/features/fiche/FichePage.tsx`,

  `src/features/membres/MembresPage.tsx`, `src/features/auth/ConnexionPage.tsx`.
- Modifier `src/main.tsx` (rendre `<RouterProvider router={router}/>`), remplacer `src/App.tsx`
  (obsolète : soit le supprimer, soit le réduire au Layout).

   `/contact/:id/modifier` via `useMatch`) — seule la pill « Fiche détail » a été retirée, comme

   prototypage, ex. spec = 6px en ligne annuaire mais 5px en fiche). Normalisé à 5px (reco/alerte/info,
   pastille/triangle) et 6px (spec, losange, visuellement plus grand) de façon cohérente dans les deux
   variantes — écart de fidélité < 1px, non perceptible.
4. **Texte des compteurs de commentaires (variant `detailed`, fiche)** : reproduit littéralement le
   texte de la maquette, ex. `"2 recommandation(s)"` (le "(s)" est toujours présent même au singulier,
   comme dans le prototype — pas une vraie pluralisation grammaticale).
5. **Type de commentaire `info`** : aucun exemple concret dans la maquette (elle n'en montre que
   reco/alerte/spec) ; forme et couleur extrapolées depuis `DECISIONS.md`/`ARCHITECTURE.md` (pastille
   bleue `#1f7fd6`, bg `#e7f1fc`, comme `commentInfoBg`/`commentInfoFg` de l'écran Édition) — titre
   popover "Infos pratiques" et libellé singulier "info pratique" inventés par cohérence avec les 3
   autres types (à ajuster si un vrai texte de référence apparaît plus tard).
6. **Boutons `ui/Button.tsx`** : implémentés en `<button>` natif (accessibilité) plutôt qu'en `<div
   onClick>` comme dans le prototype — rendu visuel identique (tous les styles de la maquette sont
   explicitement reposés), seule la sémantique HTML diffère.
