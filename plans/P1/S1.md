## T1 — Dépendances, thème, polices, CSS global

### Objectif
Poser le design system (couleurs, police, styles de base) et les dépendances de fondation.

### Décision clé
Dépendances **autorisées par ce plan** (les ajouter) : `@fontsource/plus-jakarta-sans`,
`react-router-dom`. Fonts self-hostées (RGPD, pas de CDN Google). Valeurs = celles de la maquette
(`ARCHITECTURE.md` §Maquette).

### Lire / Modifier
- Créer `src/theme/tokens.ts` (objets TS : `colors`, `radii`) et `src/theme/global.css`
  (variables CSS `--*`, reset léger, `body` en Plus Jakarta Sans, fond `#efece5`).
- Modifier `src/main.tsx` (importer `src/theme/global.css` + les poids fontsource utilisés : 400/500/600/700/800).
- Modifier `package.json` (deps) ; `src/index.css` peut être supprimé/vidé au profit de `global.css`.

### Étapes
1. `npm i @fontsource/plus-jakarta-sans react-router-dom`.
2. `tokens.ts` : reprendre les hex du design system (primaire teal `#0f9f8e`→bleu `#1f7fd6` ; sémantiques
   secteur1/secteur2/ame/vad/newpatients ; commentaires reco/alerte/spec/info ; neutres `#efece5`,
   `#fff`, bordures `#efe9dc`/`#e6e2d8`, textes `#1c2024`/`#8b8578`/`#a39c8e`).
3. `global.css` : exposer ces couleurs en variables CSS, reset (`*{box-sizing}`, `body{margin:0}`),
   `a{color:#1f7fd6}` hover `#0f9f8e`.
4. Importer `@fontsource/plus-jakarta-sans/{400,500,600,700,800}.css` + `global.css` dans `main.tsx`.

### Validation
- Auto (bloque le commit) : `npm run build` → succès ; `npm run typecheck` → 0 erreur.
- Humain (visuel) : fond beige + police appliquée → reporter dans `VALIDATION.md`.

### Si bloqué
Un poids fontsource manquant → n'importer que les poids réellement dispo dans le paquet ; ne pas ajouter d'autre dép.

## T2 — Routing + Layout + pages stub

### Objectif
Mettre en place `react-router-dom`, la barre de navigation (maquette) et une page stub par écran, pour
que le routing soit **définitif** (S3–S7 ne touchent que leur propre dossier de feature).

### Décision clé
Routes finales : `/connexion` · `/` (annuaire) · `/contact/:id` (fiche) · `/nouveau` · `/contact/:id/modifier`
· `/impression` · `/membres`. Layout = barre du haut (dégradé teal→bleu) avec logo « MSP Ménilmontant »,
pills de nav, indicateur « N sélectionné(s) → Imprimer » (props, pas de state global ici), « + Ajouter »,
pastille profil. La barre est un composant présentational ; l'état de sélection viendra plus tard (props).

### Lire / Modifier
- Créer `src/app/router.tsx` (createBrowserRouter + routes → pages stub), `src/app/Layout.tsx` (TopBar + `<Outlet/>`).
- Créer les **pages stub** (placeholder « À câbler ») dans leurs dossiers de feature :
  `src/features/annuaire/AnnuairePage.tsx`, `src/features/fiche/FichePage.tsx`,
  `src/features/edition/EditionPage.tsx`, `src/features/impression/ImpressionPage.tsx`,
  `src/features/membres/MembresPage.tsx`, `src/features/auth/ConnexionPage.tsx`.
- Modifier `src/main.tsx` (rendre `<RouterProvider router={router}/>`), remplacer `src/App.tsx`
  (obsolète : soit le supprimer, soit le réduire au Layout).

### Étapes
1. Router avec route parente `Layout` (barre visible sur les écrans app) et `/connexion` hors Layout.
2. `Layout.tsx` : reproduire la barre du haut de la maquette (couleurs/paddings des styles inline).
   Pills actives via `NavLink` (état actif = fond `rgba(255,255,255,.28)`).
3. Pages stub : un composant minimal `<div>Écran X — à câbler</div>` centré, exporté par défaut.
4. `main.tsx` : `RouterProvider`. Supprimer l'ancien contenu placeholder d'`App.tsx`.

### Validation
- Auto : `npm run build` + `npm run typecheck` OK ; navigation entre routes fonctionne (pages stub s'affichent).
- Humain : barre du haut conforme à la maquette (dégradé, pills) → `VALIDATION.md`.

## T3 — Composants partagés (Badge, CommentIcons, Avatar, primitives de formulaire)

### Objectif
Créer les composants réutilisés par plusieurs écrans, fidèles à la maquette.

### Décision clé
Composant **`CommentIcons`** central : reçoit, par type (`reco`/`alerte`/`spec`/`info`), le **compteur**
et la **liste de commentaires** ; n'affiche une icône **que si count > 0** ; popover au survol (desktop)
/ tap (mobile). Formes : reco = pastille verte `#3aa876` · alerte = triangle orange `#d3843d` · spec =
losange violet `#7a6ec9` · **info = pastille bleue `#1f7fd6`** (4ᵉ, cf. décision). Réutilisé tel quel
sur les lignes d'annuaire (S3) et la fiche (S4).

### Lire / Modifier
- Créer sous `src/components/` : `Badge.tsx` (variantes secteur1/secteur2/ame/vad/newpatients),
  `CommentIcons.tsx` (+ sous-composant popover), `Avatar.tsx` (cercle dégradé, initiales optionnelles),
  `StarToggle.tsx` (« dans ma liste »), et `ui/` : `Button.tsx`, `TextField.tsx`, `Select.tsx`,
  `Section.tsx` (bloc repliable `<details>` de la maquette).
- Types locaux des props uniquement (pas les types métier de S2) — ex. un type `CommentType = 'reco' | 'alerte' | 'spec' | 'info'` peut vivre dans `src/components/CommentIcons.tsx` en attendant S2, ou dans `src/theme`.

### Étapes
1. `Badge` : mapper variante → couleur/texte/fond (valeurs maquette). 
2. `CommentIcons` : rendu conditionnel par type, formes exactes, compteur, popover (réutiliser le markup
   de la maquette lignes 100-155 et 200-249).
3. Primitives `ui/` : styles des inputs/labels/sections de la maquette (édition, lignes 283-386).
4. Exporter chaque composant par défaut ou via un `index.ts` de barrière si utile.

### Validation
- Auto : `npm run build` + `npm run typecheck` OK.
- Humain : rendre une page de démo temporaire n'est PAS demandé ; la validation visuelle se fera quand
  S3/S4 consommeront les composants. Noter dans `VALIDATION.md` « composants partagés — à valider via S3/S4 ».

### Si bloqué
Une forme d'icône difficile → reproduire exactement le CSS de la maquette (triangle via bordures, losange
via `rotate(45deg)`), ne pas improviser une autre forme.
