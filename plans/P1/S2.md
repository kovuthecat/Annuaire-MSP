## T5 — Couche d'accès (chargement, mutations, provider)

### Objectif
Fournir aux écrans une couche d'accès simple : un chargement unique + des mutations, conformément à la
décision « recherche/filtres côté client ».

### Décision clé
**Un seul chargement** (`loadDirectory`) récupère tout (contacts, commentaires, `list_entries` du membre,
membres), compose des `ContactWithMeta[]` (commentaires groupés par type + compteurs, `starred`/`isMine`,
noms d'auteurs résolus). Les écrans filtrent/recherchent **en mémoire**. Mutations = écritures Supabase
puis rechargement (ou mise à jour locale). Pas de React Query (garder les dépendances minimales).

### Lire / Modifier
- Créer `src/data/directory.ts` : `loadDirectory()` ; mutations `createContact`, `updateContact`,
  `deleteContact`, `addComment`, `deleteComment`, `adoptContact`, `unadoptContact`.
- Créer `src/data/search.ts` : `filterContacts(contacts, query, filters)` — **fonction pure**, recherche
  tolérante (accents/casse) sur nom/prof/structure/ville/tags **et texte des commentaires** ; helper
  `findSimilarContacts(nom, contacts)` (détection de doublon pour S5).
- Créer `src/data/auth.ts` : `getSession`, `signInWithPassword(email, pwd)`, `signOut`,
  `onAuthStateChange`, `updatePassword`, `getCurrentUserId` (l'UI d'auth complète = S7).
- Créer `src/data/DirectoryProvider.tsx` + `useDirectory()` : contexte qui charge une fois (après auth),
  expose `{ contacts, members, loading, error, reload, ...mutations }`.
- Modifier `src/main.tsx` : envelopper l'arbre du routeur avec `<DirectoryProvider>` (le provider gère
  l'absence de session sans planter — RLS renvoie `[]` tant qu'on n'est pas connecté).

### Étapes
1. `loadDirectory` : requêtes en parallèle, composition des `ContactWithMeta` (grouper les commentaires
   par `type`, calculer `counts`, `starred` = présence dans `list_entries`, `isMine` = `created_by===uid || starred`).
2. `filterContacts` : normaliser (minuscules + retrait des diacritiques) requête et champs ; inclure les
   textes de commentaires dans la cible de recherche.
3. Mutations : insert/update/delete via le client ; `updated_by`/`updated_at` sont posés par le trigger SQL.
4. `DirectoryProvider` : `useEffect` de chargement, `reload()` après mutation.

### Validation
- Auto : `npm run build` + `npm run typecheck` → 0 erreur.
- Manuel (après S7 login) : `loadDirectory` renvoie sans erreur ; une création de contact apparaît. → `VALIDATION.md`.

### Si bloqué
Composition `ContactWithMeta` ambiguë → s'en tenir à la forme décrite en T4 ; ne pas inventer d'autres champs.
